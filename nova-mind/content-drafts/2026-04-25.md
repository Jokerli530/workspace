### 小红书版

**标题：🔥 Andrej Karpathy 的「AI 编码原则」开源了！一个文件让 Claude 从“笨助手”变“神队友”**

**正文：**

家人们，我最近被一个 GitHub 项目震撼到了！就是那个 AI 大佬 Andrej Karpathy 的编码原则，被大神 `forrestchang` 做成了一个单文件 `CLAUDE.md`，直接在 GitHub 上爆火，8 万多星！

说实话，用 Claude Code 写代码最怕什么？不是它不会写，而是它**太爱“自由发挥”**了！动不动就给你重构整个函数，或者加一堆你根本不需要的“优化”，最后 diff 看得人血压飙升。

这个项目简直就是“AI 编码行为矫正器”。它只干一件事：告诉 AI 四条铁律：
1.  **先思考，再动手**：别瞎猜需求，想清楚再写。
2.  **简洁是美德**：别整花里胡哨的，能一行搞定别写十行。
3.  **外科手术式修改**：让你改 A，你就别碰 B、C、D，每行改动都得有“正当理由”。
4.  **目标驱动**：别跟我说“我优化了”，直接写个测试证明它更快了。

用了之后，最大的感受就是：**AI 终于像个靠谱的同事了，而不是一个急于表现的新人。** 它不再自作主张，而是严格按照你的指令执行，代码 diff 干净得像刚擦过的黑板。

强烈建议所有用 AI 写代码的朋友都去试试，把你的 `CLAUDE.md` 文件替换成这个，你会回来谢我的！
