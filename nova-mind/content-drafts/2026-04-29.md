### 小红书版

**标题：🔥 别再手撕APK了！这个AI技能5分钟扒光App所有API接口**

家人们，谁懂啊！做竞品分析或者接盘老项目，最怕遇到没有源码的APK。以前要么跪求后端大哥，要么自己反编译看得头昏眼花。

最近挖到一个硬核开源项目，直接给Claude Code装上一个“逆向工程”技能，简直是开发者的外挂！

**为什么值得关注？**

1.  **它不只是反编译，是“挖API”**：普通的反编译工具（比如jadx）只是把Java代码给你看。但这个Skill不一样，它会自动扫描代码，把Retrofit、OkHttp的调用、硬编码的URL、认证方式全部提取出来，直接给你输出一份结构化的API文档。从“看代码”到“懂接口”，一步到位。

2.  **双引擎去混淆，更准**：现在的App都上了ProGuard/R8混淆，代码长得跟鬼一样。它同时用jadx和Fernflower两个反编译器对比分析，交叉验证，大大提高了API提取的准确率。这招“多视角验证”的思路，做技术的都懂有多香。

3.  **还能追踪调用链**：它不只是孤立地找端点，还会从Activity/Fragment出发，沿着ViewModel→Repository→HTTP调用链一路追踪，告诉你这个API是在哪个页面、什么场景下被调用的。这信息量，直接拉满。

简单说，有了它，你离“App透明化”就差一句“Decompile this APK”的距离。

### Twitter/X 版

Reverse engineering an Android APK just got a massive upgrade.

`SimoneAvogadro/android-reverse-engineering-skill` is a Claude Code skill that doesn’t just decompile code. It semantically extracts HTTP API patterns (Retrofit, OkHttp, hardcoded URLs) and traces the call flow from Activity to network layer.

The key insight? It uses a **dual-engine (jadx + Fernflower) cross-validation** strategy to combat ProGuard/R8 obfuscation. It’s not about seeing the code; it’s about understanding the API contract.

This is a blueprint for how AI agents can move from “reading code” to “understanding system behavior.”
