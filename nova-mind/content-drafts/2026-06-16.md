### 小红书版

**标题：😱 AI 编程到一半崩溃了？别慌，这个 23k+ Star 的开源项目能救你！**

**正文：**
家人们，谁懂啊！用 AI 写代码最怕什么？不是它写得烂，而是写到一半，突然 `/clear` 或者崩了，然后它像个失忆的鱼一样，前功尽弃，一切重来！😭

最近挖到了一个宝藏开源项目 **Planning-with-files**，简直是 AI 编程界的“存档大法”！

它的思路特别绝：**把规划写在文件里**。就是几个普通的 Markdown 文件（`task_plan.md`、`findings.md`、`progress.md`），却能让你的 AI Agent 拥有“永久记忆”。

不管你是手滑清空了对话，还是电脑突然崩溃，只要重新启动 Agent，它读一下这几个文件，就能无缝接上之前的进度，接着干！再也不用重复解释需求了。

更厉害的是它有个 **“完成门控”** 机制，就像给 AI 设了个“闯关游戏”，必须把计划里的任务全部打勾，才能进行下一步。完美防止 AI 偷懒跳步，或者假装任务完成了。

这个项目在 GitHub 上已经 23k+ Star 了，能兼容 Claude Code、Codex 等 60 多种 AI 编程工具。如果你也经常用 AI 写长代码、做复杂项目，这玩意儿绝对是你的救星！

### Twitter/X 版

AI coding agents 最大的痛点不是写不出代码，而是记不住上下文。

`planning-with-files` 用最朴素的方式解决了这个问题：把任务状态写在磁盘上。

三个 Markdown 文件 (`task_plan.md`, `findings.md`, `progress.md`) 构成了 Agent 的“持久化大脑”。无论 `/clear` 还是崩溃，重启后都能无缝恢复。

最聪明的设计是 Completion Gate：强制 Agent 在完成所有计划步骤前不能继续，杜绝了“假装完成”的幻觉。

23k+ Stars 不是没道理的。这才是 Agent 基建该有的样子。
