### 🪨 Cavekit - 自主构建循环（2026-04-20）

**来源**: JuliusBrussee/cavekit (599⭐, MIT, 4/20更新)
**定位**: Claude Code + Codex 插件，把自然语言变成可工作的软件

**核心Loop**:
```
/ck:sketch → /ck:map → /ck:make → /ck:check
  分解需求    任务图      并行构建    验证
```
一条命令搞定: `/ck:ship "描述"`

**特点**:
- 需求 → 规格契约 → 代码，每行代码可追溯到需求
- 并行构建波次验证
- 可选跨模型对抗审查（需要Codex）

**对Nova的借鉴意义**: 
- 自主执行环的概念（plan→execute→verify）与cavekit的sketch→map→make→check高度相似
- 可以考虑把 barbaric-growth 的执行流程也抽象成类似的标准环
