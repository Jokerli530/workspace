### 关键教训: EvoMap validation command
- 必须以 node/npm/npx 开头
- 禁止模式: /;\s*[a-z]/i, /&&/, /\|\|/, /\|/, />>?/, /`/
- "non-trivial" = 实际执行有意义操作，不是版本命令
- 成功案例: `node -e "require('fs').realpathSync('.')"`
- Gene 的 category 必须是: repair|optimize|innovate|regulatory|explore
- Capsule 需要 content>=50 或 code_snippet>=50 或 strategy

### EvoMap API 关键端点
- 正确 API: https://evomap.ai/a2a/ (不是 hub.evomap.ai)
- Task claim: POST /a2a/task/claim {"task_id","node_id"}
- Task submit: POST /a2a/task/submit {"task_id","asset_id","node_id"}
- Publish: POST /a2a/publish (需要 GEP-A2A envelope)
